Quick Start

Clone the repository and install dependencies:

git clone https://github.com/kotrotsos/logicgrid-ai.git
cd logicgrid-ai
npm install

Create a .env.local file with your Gemini API key:

VITE_GEMINI_API_KEY=your_gemini_api_key_here

Start the dev server:

npm run dev

Open http://localhost:3000 in your browser.

Build for Production

npm run build     # outputs to dist/
npm run preview   # preview the production build

How to Play

  1. Choose a topic , type anything ("Murder Mystery at a Mansion", "Space Station Crew") or pick a suggestion.
  2. Configure , select puzzle type (Logic Grid or Einstein), grid size (3x3 to 6x6), and difficulty.
  3. Generate , click the arrow button. Gemini creates the puzzle in a few seconds.
  4. Read the clues in the left sidebar. Check them off as you use them.
  5. Fill the grid:
    • Left click to mark a cell as true (green circle = these items match)
    • Right click to mark as false (red X = these items don't match)
    • Click again to clear
  6. Navigate: drag to pan, scroll to zoom, or use the reset view button.
  7. Check your work at any time. If the grid is fully and correctly filled, you win.
  8. Reveal the solution if you get stuck (this ends the game).

Puzzle Types

Standard Logic Grid

The classic format. You're given N categories each with N items. Using only the clues, determine which item from each category belongs together. Every combination of two categories forms a sub-grid in the L-shaped layout.

Einstein Puzzle

Adds a mandatory "Position" category (numbered 1 through N). Clues use spatial language: "directly left of", "next to", "between". Available in 4x4 and 5x5 sizes only.

Components

TopicSelector

Landing page form. Local state for topic, difficulty, puzzle type, and grid size. Includes five suggestion chips for quick starts. Einstein mode auto-restricts grid size to 4x4 or 5x5.

LogicGrid

The core puzzle interface, entirely rendered on an HTML5 Canvas with device-pixel-ratio support. Computes the classic L-shaped layout from N categories using computeLShapeLayout(), producing C(N,2) sub-grid blocks in a staircase arrangement.

Supports pan (drag), zoom (scroll wheel, 0.2x to 4.0x), left-click for true, right-click for false, and hover cross-highlighting of headers.

Clues

Left sidebar showing the story, goal, and a numbered clue checklist. Each clue is clickable to mark as used (strikethrough). Includes a "Get Hint" button that calls Gemini.

ResultsModal

Shown after solving or revealing. Displays the full solution as a roster table with one row per entity. Shows elapsed time, whether self-solved or revealed, and per-row match indicators (green check or red X) when self-solved.

ExportSettingsModal

Controls grid visual settings: border thickness (0.5-5px), logo toggle, and five color pickers (background, cell, header, thick borders, cell borders). Includes a reset-to-defaults button.

ExportPreviewModal

Shows a live SVG preview of the grid as it will appear when exported. Uses DOMParser for safe SVG injection. Export triggers simultaneous SVG and Markdown file downloads.

ThinkingScreen

Full-screen loading animation shown during puzzle generation. Displays an animated 2x2 grid of colored squares with rotating status messages.

Button

Reusable button component with four variants: primary (green), secondary (white with border), ghost (no background), and danger (red hover). Supports icon and loading spinner props.

Services

geminiService.ts

generatePuzzle(topic, difficulty, gridSize, puzzleType)

  1. Loads the prompt template from prompt.md and fills in template variables (topic, difficulty, category count, item count, clue range, Einstein instructions).
  2. Calls Gemini 3.1 Pro with thinkingBudget: 8192 and structured JSON output using a response schema.
  3. Validates the response (category count, item count per category).
  4. Converts the solution from array-of-arrays to array-of-objects keyed by category name.
  5. Shuffles item order within each category using Fisher-Yates to prevent diagonal solutions.

getHint(puzzle, gridState)

Sends the full puzzle and current grid state to Gemini and asks for a single logical deduction hint. Returns plain text. Falls back to a generic message on error.

exportService.ts

generateSVG(categories, gridState, settings) , builds a complete SVG string mirroring the canvas layout with all grid settings applied.

generateMarkdown(puzzle) , produces a Markdown document with the title, story, goal, clues, and category table.

exportPuzzle(puzzle, gridState, settings) , triggers simultaneous downloads of both files.

Data Model

All types are defined in types.ts.

PuzzleData

{
  title: string;
  story: string;           // 2-4 sentence narrative
  goal: string;            // Starts with "Determine..."
  categories: Category[];  // { name: string, items: string[] }
  clues: string[];
  solution: Record<string, string>[];
  puzzleType: 'standard' | 'einstein';
  gridSize: { categoryCount: number, itemCount: number, label: string };
}

GridState

A flat Record<string, CellValue> where keys are canonical pair identifiers:

// Key format: two "Category:Item" pairs, sorted and joined with "|"
"Dogs:Max|Owners:Anita" -> "true"
"Colors:Blue|Dogs:Max"  -> "false"

CellValue is one of "empty", "true", or "false".

GridSettings

{
  borderThickness: number;   // default: 2
  showLogo: boolean;         // default: true
  backgroundColor: string;   // default: #f1f5f9
  cellColor: string;         // default: #ffffff
  headerColor: string;       // default: #f1f5f9
  borderColor: string;       // default: #1e293b
  cellBorderColor: string;   // default: #e2e8f0
}

Solution Checking

checkRelationship(catA, itemA, catB, itemB) determines if two items belong to the same solution row using two strategies:

  1. Key-based: matches category names to solution keys, then compares values in the same row.
  2. Value-only fallback: checks if both values co-occur in any row regardless of key names.

Both strategies normalize with String(v).trim().toLowerCase().

Prompt Engine

The Gemini prompt is stored in services/prompt.md and loaded as a raw string at build time via Vite's ?raw import.

{{difficulty}}Easy / Medium / Hard

Prompt Structure

The prompt includes difficulty-specific rules, clue quality constraints (precision, atomicity, no redundancy, category coverage), anti-failure guardrails (bridge clues, anchor limits), a 6-step verification protocol, a clue toolbox with templates, and three full worked examples.

To modify puzzle generation behavior, edit services/prompt.md directly. No code changes needed.
